🧠 DESIGN PHILOSOPHY
─────────────────────────────────────────────────────────────────────────────
Single Polymorphic Table Design
  ✅ Avoid joins
  ✅ Flexible querying
  ✅ Dashboard-friendly
  ✅ Type-safe validation


📊 FINAL TABLE STRUCTURE
─────────────────────────────────────────────────────────────────────────────

CREATE TABLE metrics (
  id                    INTEGER PRIMARY KEY
  user_id               INTEGER FK
  measurement_type      TEXT          [glucose_fasting|glucose_postmeal|glucose_random|hba1c]
  measurement_category  TEXT          [glucose|hba1c]
  value                 FLOAT         [0.1-50 glucose | 4-15 hba1c]
  unit                  TEXT          [mmol/L|%]
  measured_at           DATETIME
  created_at            DATETIME
  note                  TEXT
  status                TEXT          [low|normal|warning|danger|prediabetes]
  estimated_hba1c       FLOAT         [from 90-day glucose avg OR NULL]
)


🔑 KEY DESIGN DECISIONS
─────────────────────────────────────────────────────────────────────────────

1️⃣  MEASUREMENT_TYPE (Polymorphic Key)
   - glucose_fasting: Early morning (8-10h fast)
   - glucose_postmeal: 2h after eating
   - glucose_random: Any time (no fast needed)
   - hba1c: 3-month glucose average (%)

2️⃣  MEASUREMENT_CATEGORY (Quick Filter)
   - glucose: Filter all glucose types quickly
   - hba1c: Filter HbA1c only
   - NO LIKE queries needed

3️⃣  VALUE & UNIT (Type-Dependent)
   - Glucose: 0.1-50 mmol/L, daily readings allowed
   - HbA1c: 4.0-15.0 %, quarterly readings
   - Different scales, ranges, frequencies

4️⃣  STATUS (Pre-calculated)
   - Glucose: low | normal | warning | danger
   - HbA1c: normal | prediabetes | danger
   - Cached at insert time (no recalc on query)

5️⃣  ESTIMATED_HbA1c (Calculated Field)
   - Formula: HbA1c = (0.0915 × Avg_Glucose_mmol/L) + 2.15
   - From 90-day rolling average of glucose readings
   - NULL for actual HbA1c readings
   - Show users: "Your estimated HbA1c: 6.8%"


📈 GLUCOSE → HbA1c CONVERSION
─────────────────────────────────────────────────────────────────────────────

When user adds glucose reading:
  1. Insert into metrics table
  2. Calculate status (danger/warning/normal/low)
  3. Query last 90 days of glucose readings
  4. Calculate average
  5. Apply formula: HbA1c = (0.0915 × avg) + 2.15
  6. Update estimated_hba1c field
  7. User sees both reading AND estimated HbA1c on dashboard

Example:
  User adds: Fasting 7.2 mmol/L
  System calculates:
    ├─ Status: danger (7.2 > 7.0)
    ├─ 90-day avg: 8.3 mmol/L (from 45 readings)
    ├─ Estimated: (0.0915 × 8.3) + 2.15 = 6.79%
    └─ Display: "7.2 mmol/L (Danger) | Estimated HbA1c: 6.79%"


🏃 COMPLETE WORKFLOW
─────────────────────────────────────────────────────────────────────────────

ADD GLUCOSE:
  1. User enters type + value + time + note
  2. Frontend validates (0.1-50 range)
  3. POST /api/v1/metrics
  4. Backend:
     - Validate with Zod
     - Set category='glucose', unit='mmol/L'
     - Calculate status
     - Get 90-day glucose avg
     - Calculate estimated HbA1c
     - INSERT into metrics with all fields
  5. Return with calculated fields
  6. Dashboard shows latest + estimated HbA1c

ADD HbA1c:
  1. User/doctor enters HbA1c value
  2. Frontend validates (4-15 range, %)
  3. POST /api/v1/metrics
  4. Backend:
     - Validate with Zod
     - Set category='hba1c', unit='%'
     - Calculate status
     - estimated_hba1c = NULL (actual reading)
     - INSERT into metrics
  5. Dashboard shows actual HbA1c
  6. Show comparison vs estimated
